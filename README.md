2. Постановка задачи
Разработать десктопное WPF-приложение для автоматизации учета деятельности спортивной секции, включающее:
Управление списком участников (добавление, удаление, сохранение)
Ведение журнала посещаемости
Управление тренерским составом
Составление расписания занятий
Фильтрацию и отображение расписания
Сохранение данных в текстовые файлы

3. Пользователи и бизнес-процессы
Основные пользователи:
Администратор спортивной секции
Бизнес-процессы:
Управление участниками:
Добавление новых участников
Удаление участников
Загрузка/сохранение данных
Учет посещаемости:
Отметка присутствия участников
Просмотр статистики посещений
Фильтрация по датам
Управление тренерами:
Добавление/удаление тренеров
Редактирование информации о тренерах
Учет специализации и опыта
Составление расписания:
Планирование занятий по дням недели
Привязка тренеров к занятиям
Управление группами
Фильтрация расписания (по тренеру, по дню)


6. Инструменты разработки
Язык программирования: C#
Фреймворк: .NET Framework / .NET Core (WPF)
IDE: Visual Studio 2022+
Система контроля версий: Git
Формат данных: Текстовые файлы (CSV-подобный формат)
Технологии: XAML, MVVM-паттерн (частично)

7. Интерфейс приложения
Приложение использует интерфейс с вкладками (TabControl):
1. Вкладка "Участники" (Основная):
DataGrid со списком участников (ФИО, телефон, email)
Панель управления: кнопки "Удалить", "Сохранить", "Загрузить"
2. Вкладка "Посещаемость":
Выбор участника (ComboBox)
Выбор даты (DatePicker)
Кнопка отметки посещения
Статистика (общее количество участников, посещений сегодня)
DataGrid журнала посещений
3. Вкладка "Тренеры":
DataGrid со списком тренеров (ФИО, телефон, email, специализация, стаж)
Панель управления: "Добавить тренера", "Удалить тренера", "Редактировать"
4. Вкладка "Расписание":
Форма добавления занятия: день недели, время, тренер, группа
Панель фильтров: "Все", "Сегодня", "По тренеру"
DataGrid расписания
5. Вкладка "Добавить участника":
Форма ввода: ФИО, телефон, email
Кнопка "Добавить участника"

8. Структура данных
Файлы данных (текстовые):
members.txt - список участников
Формат: GUID|ФИО|Телефон|Email
attendance.txt - журнал посещаемости
Формат: GUID_участника|ФИО|Дата|Статус
coaches.txt - список тренеров
Формат: GUID|ФИО|Телефон|Email|Специализация|Стаж
schedule.txt - расписание занятий
Формат: GUID|ДеньНедели|Время|GUID_тренера|ФИО_тренера|Группа|Продолжительность

10. Структура кода
SportSectionApp/
│
├── MainWindow.xaml              # XAML разметка интерфейса
├── MainWindow.xaml.cs           # Логика работы приложения
│
└── Классы данных:
    • Member                     # Участник секции
    • AttendanceRecord           # Запись о посещении
    • Coach                      # Тренер
    • ScheduleItem               # Элемент расписания

Основные методы в MainWindow.xaml.cs:
LoadData() - загрузка всех данных из файлов
SaveMembers(), SaveAttendance(), SaveCoaches(), SaveSchedule() - сохранение данных
UpdateUI() - обновление интерфейса
UpdateStatistics() - расчет и отображение статистики
UpdateScheduleView() - фильтрация и отображение расписания
